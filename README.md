# 🧬 Protein Design Evaluation using ESM2

A minimal, end-to-end AI workflow to evaluate native and designed protein sequences using protein language model embeddings.

---

## 🚀 Why this project exists

Modern protein engineering is no longer based only on structure and energy calculations.
AI models trained on millions of protein sequences (protein language models) can capture patterns that are difficult to encode manually.

This project demonstrates a simple but powerful idea:

> **Can we use an AI model to check whether designed protein sequences still “look like” the native protein?**

Instead of relying only on mutation counts or structural predictions, we introduce a **learned representation-based evaluation layer** using ESM2.

---

## 🧠 Key idea

Each protein sequence is converted into a numeric vector (embedding) using the ESM2 model.

We then:

* represent each sequence as a point in high-dimensional space
* measure similarity between sequences using cosine similarity
* rank designed proteins based on closeness to the native sequence

👉 This gives a **data-driven similarity signal** learned from real protein sequence distributions.

---

## 🔬 What this project demonstrates

This repository shows a minimal version of a modern protein design workflow:

```
Protein sequence → AI embedding → Similarity comparison → Design ranking
```

It acts as an **AI-based evaluation layer** that can complement:

* structure prediction (e.g., AlphaFold / ColabFold)
* stability estimation (e.g., FoldX)
* sequence design (e.g., ProteinMPNN)

---

## ⚙️ Workflow

1. Load native and designed sequences from a FASTA file
2. Convert sequences into embeddings using ESM2
3. Aggregate residue-level embeddings into sequence-level vectors
4. Compute pairwise cosine similarity
5. Rank designs based on similarity to the native sequence

---

## ⚠️ Important limitation

This method does **not directly predict**:

* protein stability
* folding accuracy
* biological function

Instead, it measures:

> **how “natural” or “native-like” a sequence appears to a protein language model**

It should be used alongside:

* structural validation
* energetic scoring
* experimental data

---

## 💡 Why cosine similarity?

Cosine similarity compares the **direction** of embedding vectors:

[
\text{similarity}(x, y) = \frac{x \cdot y}{||x|| , ||y||}
]

This makes it ideal for comparing learned representations, independent of scale.
